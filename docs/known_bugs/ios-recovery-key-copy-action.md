# iOS: Recovery Key View Has Copy Action

Status: Fixed.

## Summary

The iOS recovery setup/confirmation UI previously displayed a recovery key but
did not provide a copy action. Recovery keys are long and error-prone to
transcribe; the user needs an explicit, safe copy button.

## Original user-visible symptom

- User starts Matrix SDK recovery setup on iOS.
- The generated recovery key is shown.
- There is no obvious button to copy it to the clipboard.

## Expected behavior

- The recovery key display should include a clear copy action.
- The copy action should provide immediate feedback, for example "Copied".
- The key must still not be printed to logs or persisted outside the intended UI
  state.
- The UI should remind the user to save the key, but should not expose it in
  telemetry, debug output, or screenshots generated by automated tests unless
  explicitly redacted.

## Investigation notes

- Start in `apple/Sources/Shared/ViewModels/DeviceVerificationViewModel.swift`
  and the account/device verification SwiftUI views.
- Locate where `displayedRecoveryKey` is rendered.
- Add the UI action at the view layer while leaving Matrix SDK recovery setup in
  `MatrixDeviceVerificationService` / `MatrixRustSDKAdapter`.
- Use the platform clipboard API through a small platform abstraction if the
  same UX is implemented for iOS and macOS.

## Implementation requirements

- Do not log recovery keys.
- Do not save recovery keys in Keychain or UserDefaults as part of a copy button.
- Do not add a shortcut that resets identity or recovery state.
- Do not bypass Matrix SDK recovery APIs.
- Prefer a reusable platform copy helper so iOS and macOS behavior stays
  consistent without putting SDK logic in SwiftUI views.

## Verification plan

- Use a dedicated disposable recovery smoke account if live recovery mutation is
  required. Do not run recovery mutation against the admin account.
- Build iOS with the normal Matrix Apple verification chain.
- Exercise recovery setup until a generated recovery key appears.
- Tap copy.
- Paste into a local text field or Notes-like scratch target and confirm the
  pasted value matches the displayed key.
- Confirm app logs contain no recovery key material.

## Acceptance criteria

- iOS recovery key UI has an explicit copy button.
- Copy feedback is visible.
- Recovery key is not logged, persisted, or printed by live smoke output.

## Resolution

- The shared Matrix recovery key surface now exposes a `Copy Recovery Key`
  action when a generated key is displayed.
- The button uses the iOS pasteboard through a small shared platform helper and
  switches to `Copied` after the user taps it.
- The copy path does not log the key, write it to Keychain, write it to
  UserDefaults, or touch Matrix SDK recovery state.
